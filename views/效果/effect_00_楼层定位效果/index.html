使用

基本示例

和前几个小点心相同的 API 风格，简单，尽可能简单。

const floorA = new Floor({
container: '#floor-A',
baseline: 0.2, // 基准线，作为判断 active floor 的依据，
baselineDebug: true, // 是否显示基准线
onFloorChange(){
  $('#floor-A-num').html(this.activeIndex)

不同于通用弹窗和底部浮现弹窗，楼层定位对 DOM 结构和 CSS 样式有了更多的要求。没办法，这个效果涉及到滚动，需要改变原生滚动的行为，而且需要考虑到各个平台系统的兼容性。因此一定的 DOM 结构和 CSS 样式要求是不可避免的。

期望的 DOM 结构：

<div id="your-floors-container">
<div id="your-floors-wrapper">
  <div class="your-floor-item">
        .
        .
        .
  <div class="your-floor-item">
</div>
</div>

如果你用过 Swiper.js，对这种结构一定不会陌生，没错，这是产生滚动的必须条件。请使用这种结构，否则可能会导致莫名其妙的 bug。

期望的 CSS 样式：

#your-floors-container{
position: relative;
overflow: hidden;
}

这就是所需要的全部东西。注意哟，overflow 的属性值是 hidden 而不是 auto 或其他，因为这里使用的是模拟滚动，而不是原生滚动。也就是在用户滑动的时候通过操控元素的 transform 和 transition 去移动它，使其看起来像滚动一样。之所以这么麻烦滴去模拟，是因为原生滚动在各个平台（安卓和 IOS 以及 PC 端各种浏览器）表现不一致且很难控制，模拟滚动可以提供给开发者更精细更完善的控制权以及更一致的 UI 表现。

基于 iscroll.js 实现

需要提前声明的是，这个效果在内部使用了 iscroll.js 来辅助实现。但并不期望你对 iscroll 有什么了解，因为本插件已将其封装的足够简单。

onFloorChange

onFloorChange 是本小点心唯一的回调，也是本插件最关键的功能。onFloorChange 的触发时机是仅在【焦点楼层】发生变化时。

雷区

这里有一些在使用时容易导致 bug 的地方，其中的大部分应该都不容易碰到，但为了避免你在遇到它们时大骂“沃特法克”，请仔细阅读它们。

在 DOM 加载完毕之后实例化

由于基于 iscroll.js，而 iscroll 需要拿到容器的精确尺寸并且会缓存起来。如果 DOM 未加载完毕就实例化，很可能会导致行为异常。

在与 MVVM 框架混合使用时尤其注意这一点，例如当结合 Vue 使用时，请在 Vue 挂载到 DOM 之后再进行实例化，你可以在mounted生命周期里去new Floor()^_^

容器位置或尺寸发生变化后调用refresh

依然是 iscroll.js 需要的，这事真不赖我。iscroll 会在实例化时缓存容器位置和尺寸信息，当发生变化后，这些缓存的数值就过期了。因此必须手动调用 refresh 方法告诉 iscroll 重新计算。

API

和其他小点心保持了统一的 api 风格，因为我们都是小点心家族咯^_^

new Floor(options)

用于实例化一个【楼层定位】并返回该实例的引用。

options
  • container：
    必选，楼层容器，即楼层的可视窗口，可传入id，class等css选择器，强烈推荐传入id，因为这很符合面向对象哲学^_^；
  • baseline：
    可选，基准线，也就是滑到哪才算是切换了楼层，这个值是相对于楼层容器（也就是传入 container 元素）高度的百分比，默认是 0.5，即滑过容器一半就算是切换了楼层。如果设置成 0.2，那就是换到距离容器顶部 20% 高度时即算是切换了楼层。因此这个选项的取值区间最好是 [0, 1]；
  • baselineDebug：
    可选，是否展示基准线，即是否显示前面几个示例中的那条虚线。默认是不显示的，你可以将其设为 true 将这条线显示出来，这应该仅做调试用；
  • itemClass：
    可选，有时你可能并不想 wrapper 中的每个子元素都作为一个楼层，这就是 itemClass 选项的作用，你可以传入一个 CSS 选择器，Floor 将只把筛选出来的元素作为楼层。默认会将 wrapper 下的所有子元素都作为楼层来处理；
  • iscroll：
    可选，正如前面所说的，Floor 为了兼容性和用户体验，选择了基于 iscroll.js 来实现，因此这个选项用于控制 IScroll。如果你对 iscroll.js 一无所知，那就太好了，完全忽略这个选项吧。如果你是个 iscroll.js 专家，想自定义 IScroll 的行为，那就把自定义项都写在这里面，实例化 IScroll 时的所有选项都可以写在这里面；
  • onFloorChange：
    可选，楼层切换回调，这个是本插件的最核心功能。调用时机是在用户滑动楼层并且导致楼层发生切换时，可根据上面的示例免费体验一下；

floor.scrollTo()

将楼层滚动到指定位置。该方法支持传入不同的参数实现不同的行为。

scrollTo 支持很多种不同的传参方式，但大体使用范式如下：

floor.scrollTo(y, [opts])

y

  • 传入一个数值如 100，表示滚动至 n 个像素；
  • 传入 '+100' 或 '-50' 表示相对于现在的位置再偏移 n 个像素；
  • 传入一个 css 选择器（可以是任何 jQuery 支持的选择器如.floor:last或.floor:nth-child(3)等）或者 dom 元素或者 jQuery 对象，表示直接【滚动到指定元素】位置，这通常是最有用的。

opts

opts 可以是一个或多个参数，你可以像下面这样使用：

  • floor.scrollTo(y, [time], [easing])，time 表示动画时间，单位毫秒(ms)，默认 0。easing 表示缓冲函数，可以是以下值之一：quadratic | circular | back | bounce | elastic。
  • floor.scrollTo(y, {time: 100, easing: 'elastic', offsetY: true})，time 和 easing 同上，offsetY 用于在【滚动到指定元素】后再偏移 n 个像素，或者可以直接传 true，这样就会滚动到指定元素居容器正中位置，这是非常实用的。

floor.refresh()

IScroll 需要知道 container 和 wrapper 实际尺寸，并且在初始化时对其做了缓存。因此如果 IScroll 内的元素数量或位置大小发生了变化，你就需要告诉它重新计算，这是很蠢的。但是没办法，凑活着用吧，谁让咱依赖他呢。所以如果 container 位置或尺寸发生了变化，请手动调用 refresh 刷新一下，make iscroll happy^_^

floor.on(event, callback)

有时你可能需要在 scroll 或 scrollEnd 事件上自定义监听函数，你可以使用这个方法去绑定事件。支持的事件有：beforeScrollStart | scrollCancel | scrollStart | scroll | scrollEnd 等。
